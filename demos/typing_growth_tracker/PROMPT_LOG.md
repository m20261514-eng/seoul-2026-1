# PROMPT_LOG.md

GitHub Copilot Chat에게 입력한 프롬프트 기록. 시간 순서대로, 시행착오 포함.

> **왜 기록하나요?** Copilot이 잘 작동한 프롬프트와 실패한 프롬프트를 비교해야
> "좋은 프롬프트의 감각"이 늡니다. 코드를 베끼지 말고 프롬프트를 배우세요.

## #2 — 기능 1: CSV 업로드와 표 표시

> **Prompt**

**결과:** 동작은 했지만, 업로드 후 표만 떠서 밋밋함. 명세에 "응답자 수가 상단에 뜬다"가 있는데 빠져있음 → #3에서 보완.

## #3 — 상단 요약 메트릭 추가

> **Prompt**
> 표 위쪽에 `st.columns(4)`로 4개의 메트릭 카드를 만들어줘.
> 항목은: ① 응답자 수(행 수), ② 평균 1회차 타수, ③ 평균 3회차 타수, ④ 평균 성장률.
> 성장률은 (3회차 - 1회차) / 1회차 * 100 으로 계산해.

**결과:** 깔끔하게 4개의 메트릭이 한 줄에. 명세의 "응답자 수가 상단에 뜬다" 충족.

## #4 — 인코딩 사고 (실패 사례)

> **Prompt**
> CSV 업로드할 때 한글이 깨져. 어떻게 해결해?

**결과:** Copilot이 `encoding="utf-8"`만 추가. 그런데 엑셀에서 저장한 CSV는 CP949라서 여전히 깨짐. 명세에 "UTF-8/CP949 둘 다 대응"이라고 적었던 이유가 이거였음.

> **Prompt (수정본)**
> 한국 교사가 엑셀에서 "CSV로 저장"하면 보통 CP949 인코딩이고, 메모장에서 만들면 UTF-8이야.
> `read_csv`를 utf-8-sig → utf-8 → cp949 → euc-kr 순서로 try/except 하면서 시도하고,
> 다 실패하면 errors="replace"로 폴백하는 헬퍼 함수 `read_csv_any(file)`를 만들어줘.
> 파일을 `io.BytesIO`로 감싸서 매번 재시도할 수 있게 해줘.

**결과:** 견고해짐. **교훈: "한글이 깨져"보다 "엑셀 CSV는 CP949"라고 도메인 맥락을 주면 답이 정확해진다.**

## #5 — 입력 검증

> **Prompt**
> 사용자가 컬럼명이 다른 CSV를 올리면 앱이 그냥 죽어버려.
> 업로드된 DataFrame을 받아 다음을 검사하는 `validate(df)` 함수를 만들어줘:
> 1. 필수 컬럼(`이름`, `1회차`, `2회차`, `3회차`)이 모두 있는지
> 2. 회차 컬럼이 숫자인지 (`pd.to_numeric(..., errors="coerce")`)
> 3. 1회차에 0이나 음수가 있는지 (성장률 분모가 0이 되면 안 됨)
> 에러 메시지 리스트를 반환하고, 에러가 있으면 `st.error`로 보여주고 `st.stop()` 해줘.

**결과:** 사고를 미리 차단. **교훈: 명세 단계의 "Out of Scope"를 넘어, 입력의 엣지 케이스를 의식적으로 챙겨야 한다.**

## #6 — 기능 2: 학생 선택 + 꺾은선 그래프 (한글 폰트 사고)

> **Prompt**
> `st.selectbox`로 학생을 선택하면, 그 학생의 1·2·3회차 타수를 matplotlib 꺾은선 그래프로 그려줘.

**결과:** 한글 라벨이 모두 "□□□" 박스로 깨짐. matplotlib은 시스템 폰트 없으면 한글이 안 나옴. Streamlit Cloud는 한글 폰트 미설치.

> **Prompt (수정본)**
> matplotlib 말고 plotly로 바꿔줘. plotly는 SVG로 렌더링해서 한글이 자동으로 잘 나와.
> `plotly.graph_objects.Figure`로 학생의 1·2·3회차를 꺾은선으로 그리고,
> 각 점 위에 "120타" 식으로 값 라벨을 표시해.

**결과:** 한글 깨짐 해결. **교훈: Streamlit Cloud에서 matplotlib 한글은 별도 폰트 세팅이 필요 → plotly가 훨씬 편하다.**
